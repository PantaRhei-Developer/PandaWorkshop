## 内容の要約
### Panta Rheiが導入すべきMCP一覧（Pandasista）

Claude Code をさらに強化する MCP（Model Context Protocol） の導入を提案しました。MCPを入れることで、Claude Codeの能力は10倍・100倍になりうることから、今回おすすめのMCPについて話してくださったそうです。<br>

MCPはAnthropicが策定した拡張プロトコルで、Claude Code の CLI に組み込むことで、外部ツールやサービスと連携できるようになります。設定は .claude.json の MCP サーバーセクションに書き込むだけ。

| MCP | 機能 |
| :-- | :-- |
| Context7 | 最新ドキュメントをリアルタイム参照 |
| Serena | LSPベースのコード解析 |
| GitHub MCP | PR作成・レビューをブラウザ不要で実行。Git の基本理解後に導入推奨 |
| VoiceBox（ずんだもん）| 作業完了を音声で通知 |

一方で、「Claude Code を搭載した AI が 9秒で会社のデータベースとバックアップを消滅させた」という事例が共有されました。原因は環境整備の不備とのことで、インフラの知識とリスク対策がAI を使う上で欠かせないと改めて強調されました。<br><br>


### React初学者がHPを改変したことについて（onigiririceball）

今回の前座として、React 学習中のonigiririceballさんが、ホームページ改変タスクで実際に仕様書駆動開発に挑戦した体験を共有しました。<br>

取り組んだのは「ホームページ最下部に YouTube と Compass のロゴを追加する」というタスクでした。他のページへの影響が少なく、難易度が低めで安心して試せるという理由でした。<br>

開発フローはこの4ステップは、
① 社内マニュアルを確認 → ② Claude Code に読み込ませて仕様書を生成 → ③ 仕様書に沿って実装 → ④ レビュー担当者に確認依頼<br>

全体を通じて、エンジニアとして介入必要な場面はほとんどなく、「どの画像を使うか」といった非技術的な判断のみ人間が行うことで実装できました。このタスクにおいてエラー対応などの苦悩はなかったようです。その中でエンジニアとして必要な知識を持ちつつAIを使用するために「リーダブルコード」という書籍が推奨されていました。
